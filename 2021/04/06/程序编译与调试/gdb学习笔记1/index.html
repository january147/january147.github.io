GDB学习笔记1

在Linux中做c开发时经常会使用gdb来进行调试，本博客会介绍我经常使用的gdb命令。

基本命令

命令 说明
`b [
n 执行下一行代码
ni 执行下一条指令
finish 执行到当前函数结束
c 继续执行
set scheduler-locking [on/off] 禁止/允许线程切换
p [<variable>] 打印变量
x/[<size>][<format>] [<addr>] 查看内存数据
info b 查看断点
info registers 查看寄存器
del [<number>] 删除断点
file <filename> 加载可执行文件
add-symbol-file <filename> -s <section_name> <addr> 加载符号文件（使用-s来设置指定section的加载地址）
symbol-file 删除符号文件

图形界面

命令 说明
ctrl + x , a 切换显示方式（Terminal UI/CLI）
layout asm 切换到汇编代码界面
layout src 切换到源代码界面

反汇编

使用display每次执行命令后显示反汇编结果，并显示当前pc位置，start_address可以是符号，地址或者寄存器。

访问变量/内存/寄存器

使用p(print)来查看变量/寄存器，使用x来查看内存区域

格式字符(format) 含义
o(octal) 8进制数字
x(hex） 16进制数字
d(decimal) 10进制数字
u(unsigned decimal) 10进制无符号整数
t(binary) 二进制数字
f(float) 浮点数
a(address) 地址
i(instruction) 指令
c(char) 字符
s(string) 字符串
z(hex, zero padded on the left) 左侧使用0填充的16进制数字
长度字符(item size) 含义
b(byte) 1字节
h(halfword) 2字节
w(word) 4字节
g(gaint) 8字节

修改变量/内存/寄存器

使用print来修改变量和寄存器
