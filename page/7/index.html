ARM寄存器

系统寄存器

SPSel

该寄存器控制SP寄存器的使用

长度： 64 bit

字段 名称 说明
bits[63:1] - 未使用
bits[0] SP 0b0表示所有异常级别均使用SP_EL0
0b1表示每个异常级别使用单独的SP寄存器(SP_ELx)
该字段默认重置为0b1（使用单独的SP寄存器）

参考Arm® Architecture Reference Manual for Armv8(G.a) 447页

mpidr_el1

该寄存器用于在多处理（多核心）系统中识别当前运行的处理器。

长度: 64bit

字段 名称 说明
bits[63:8] - 参考ARM手册
bits[7:0] - 一般为处理器编号

参考Arm® Architecture Reference Manual for Armv8(G.a) 3527页

daifclr

该寄存器用于清除中断掩码（启用中断），只能写入

长度： 暂时还不确定

字段 名称 说明
bits[0] D Debug中断掩码清除，设置为1表示清除对应中断掩码（置对应掩码为0），0表示不改变中断掩码
bits[1] A SErrors中断（外设错误）掩码清除（It is called A because SErrors sometimes are called asynchronous aborts.）
bits[2] I IRQ掩码清除
bit F FIQ掩码清除

daifset

该寄存器用于设置中断掩码（屏蔽中断），只能写入

长度：暂时还不确定

字段 名称 说明
bits[0] D Debug中断掩码设置，设置为1表示设置对应中断掩码为1，0表示不改变中断掩码
bits[1] A SErrors中断（外设错误）掩码设置
bits[2] I IRQ掩码设置
bit F FIQ掩码设置

CurrentEL

该寄存器保存当前异常级别

长度: 64bit

字段 名称 说明
bits[63:4] - 未使用
bits[3:2] EL 当前异常级别
0b00表示EL0
0b01表示EL1
0b10表示EL2
0b11表示EL3
bits[1:0] - 未使用

参考Arm® Architecture Reference Manual for Armv8(G.a) 407页

clone实现

在进程地址空间不同的情况下无法实现clone，因为进程的状态（寄存器的内容）和进程地址空间是相关的，如果新创建的进程地址空间同原进程不同，则原进程寄存器内容对于新进程来说则是错误的，会导致程序无法正常运行。

例如原进程的x19寄存器保存了原进程的一个栈中的地址0x0011，而新进程的栈地址空间和原进程不同，如果复制原进程的x19寄存器则会导致新进程在使用x19寄存器访问内存时发生发生非法内存访问。

ARM中断处理

处理器中断控制

处理器中断由daifclr寄存器控制, 该寄存器共有4个有效比特，每个比特控制一类中断的状态

  • D Masks debug exceptions. These are a special type of synchronous exceptions. For obvious reasons, it is not possible to mask all synchronous exceptions, but it is convenient to have a separate flag that can mask debug exceptions.
  • A Masks SErrors. It is called A because SErrors sometimes are called asynchronous aborts.
  • I Masks IRQs
  • F Masks FIQs

中断控制器

中断控制器属于外设，通过MMIO来进行控制

local interrupt controller

本地中断控制器用于控制单个核心的中断，其包括来自全局中端控制器的中断以及其他一些本地中断（例如本地时钟中断、核心通信中断等）

对于raspi3b，其本地中断控制的pending register对应的中断如下。

Accordingly to BCM2836 ARM-local peripherals manual local interrupt controller handles 10 different interrupts: 0 - 3 are interrupts from local timer, 4 - 7 are mailbox interrupts, which are used in interprocess communication, 8 corresponds to all interrupts generated by the global interrupt controller and interrupt 9 is a performance monitor interrupt.

global interrupt controller

全局中断控制器用于控制整个系统的外设中断，全局中断控制器接收的中断会发送到指定的本地中断控制器。

gdb汇编错误

错误的原因是qemu提供给gdb的架构位i386,而32位的i386被gdb视为16位i8086的超集，即i386兼容i8086，当我们使用set architecture设置架构为i8086时就不会生效。

https://sourceware.org/bugzilla/show_bug.cgi?id=22869

https://stackoverflow.com/questions/32955887/how-to-disassemble-16-bit-x86-boot-sector-code-in-gdb-with-x-i-pc-it-gets-tr

使用set tdsec filename <file>来设置一个描述文件可以将gdb设置为i8086模式
